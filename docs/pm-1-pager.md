# SaveTicker PM 1-Pager

> 금융 뉴스를 "이해"로 바꾸는 프로토타입 — SAVE 앱 PM 포트폴리오

---

## 1. Problem (문제)

한국 금융 뉴스에는 3중 장벽이 존재합니다:

| 장벽 | 현상 | 영향 |
|------|------|------|
| **영어 원문** | 글로벌 속보는 영어로 먼저 나옴 | 한국어 번역까지 시간 지연, 의역 부재 |
| **전문용어** | VLCC, BDC, PMI, CPI 등 | 비전문가는 기사를 읽어도 "무슨 뜻인지" 모름 |
| **단편성** | 각 기사가 개별 사건으로 소비됨 | 사건 간 맥락(원인→결과)을 독자가 직접 연결해야 함 |

### 근거 데이터
- 2024 전국민 금융이해력 조사: **65.7점** (2022년 66.5점 대비 하락)
- 금융 태도 점수: **53.7점** — OECD 평균(58.0점) 미달
- 디지털 금융이해력: **45.5점** — 갈수록 중요해지지만 여전히 낮음
- **취약 계층**: 20대 청년, **70대 이상 노인**, 저소득층, 고졸 미만
- 출처: 한국은행·금융감독원 「2024 전국민 금융이해력 조사」

---

## 2. 핵심 철학: 중립적 정보 제공

> **"종목 추천은 절대 하지 않습니다."**

SaveTicker의 모든 콘텐츠는 **중립적 사실 전달**과 **맥락 제공**에 집중합니다.
사용자가 **스스로 판단할 수 있는 기반**을 만들어주는 것이 목적이지,
특정 종목을 사거나 팔라고 유도하지 않습니다.

이 철학은 SAVE 앱의 가치 — "투자의 모든 소식, 이제 한곳에서" — 와 정합합니다.
SAVE는 **정보의 속도와 접근성**을 제공하지, 투자 조언을 제공하지 않습니다.

---

## 3. Hypothesis (가설)

**단편적 속보를 시간순 스토리로 엮고, 전문용어를 쉬운 비유로 풀어주면,
금융 뉴스를 처음 접하는 사람(노인 포함)도 맥락을 이해하고 올바른 판단을 내릴 수 있다.**

검증 방법: 프로토타입을 만들어 실제 동작을 보여줌.

---

## 4. Solution (해결) — 3대 기능

### Feature 1: Story Threads (스토리 흐름)
- **Before**: 8개 속보가 개별로 나열 → "이란 공습"과 "유가 급등"의 관계를 독자가 직접 파악
- **After**: 관련 기사가 시간순 타임라인으로 묶임 → 원인→전개→영향이 한눈에

### Feature 2: Plain Language Cards (쉬운 해설)
- **Before**: "VLCC 운임 94% 급등" → 비전문가: "VLCC가 뭐지?"
- **After**: "보험 없이는 배가 못 간다" + 비유 + "나에게 어떤 영향?" → 누구나 이해
- 난이도 표시 (입문/중급/심화) + 읽기 시간 추정 + 핵심 한 줄 요약

### Feature 3: Impact Chains (영향 도미노) — 설계 완료
- 하나의 사건이 어떤 연쇄반응을 일으키는지 트리 구조로 시각화
- 예: 이란 공습 → 해협 봉쇄 → 보험 철회 → 유조선 운임 급등 → 유가 상승 → 한국 에너지 위기

## 5. SAVE 3가치 매핑

| SAVE 핵심 가치 | 대응 기능 | 구현 방법 |
|---------------|-----------|----------|
| **압도적인 속도** | Story Threads | 새 기사가 기존 스레드에 자동 편입, 실시간 타임라인 |
| **완벽한 개인화** | Plain Language Cards | 난이도별 맞춤 해설, "나에게 어떤 영향?" 개인 맥락 |
| **최고의 효율성** | Story Threads + Explainers | 맥락 파악 시간 단축, 정보 탐색 → 이해로 전환 |

---

## 6. TAM / SAM / SOM

| 구분 | 정의 | 규모 |
|------|------|------|
| **TAM** | 금융문맹 해소 대상 — 한국 성인 중 금융태도 점수 OECD 미달 계층 | ~1,800만 명 (성인 인구의 ~42%) |
| **SAM** | 모바일 금융뉴스 이용자 (30-60대, 스마트폰 뉴스 소비) | ~800만 명 |
| **SOM** | SAVE 앱 사용자 중 StoryTelling 기능 활성 비율 목표 (Year 1) | DAU의 15-25% |

특히 **노인층(60대+)** 은 금융 자산은 많지만 디지털 금융이해력(45.5점)이 가장 낮은 계층으로,
"노인도 이해할 수 있는 StoryTelling"은 TAM 확장의 핵심 레버입니다.

## 7. 측정 지표 (North Star + 보조)

| 지표 | 의미 | 목표 |
|------|------|------|
| **StoryTelling 탭 전환율** (North Star) | 기사 상세 → StoryTelling 탭 클릭 비율 | 30%+ |
| 기사 완독률 | 스크롤 depth 80%+ 도달 비율 | 기존 대비 +20% |
| 스레드 탐색 깊이 | 스레드 내 2개 이상 기사 클릭 비율 | 60%+ |
| 난이도별 체류 시간 | 입문/중급/심화별 평균 체류 시간 | 입문 > 중급 > 심화 |

## 8. Prototype Evidence

| 항목 | 구현 상태 |
|------|----------|
| Story Threads | Convex 테이블 + 쿼리 + 시드 (2스레드 × 4기사) + 타임라인 UI |
| Plain Language Cards | CRUD 완성 + 8개 한국어 해설 + 핵심 한 줄 + 어코디언 |
| Impact Chains | 온톨로지 정의 완료 (트리 구조), UI 설계 중 |
| 기술 스택 | TypeScript strict + Convex + React 19 + Capacitor 8 |
| 접근성 | 48px 터치 타겟, WCAG AA 대비, ARIA, 키보드 네비게이션 |
| 모바일 | Android 하드웨어 백 버튼, 햅틱 피드백, safe area |

## 9. PM으로서의 역량 증명

| 역량 | 증거 |
|------|------|
| **문제 발견** | SAVE 앱의 기존 구조 분석 → 3중 장벽 도출 |
| **해결책 설계** | 온톨로지 기반 4-fold 스키마 설계 (Palantir 모델 적용) |
| **직접 구현** | 풀스택 프로토타입 (백엔드 + 프론트엔드 + 모바일) |
| **사용자 관점** | 노인 접근성, 난이도별 맞춤, "나에게 어떤 영향?" |
| **중립성** | 종목 추천 없음 — 사실 전달과 맥락 제공만 |
| **데이터 기반** | OECD/한국은행 금융이해력 통계로 TAM 근거 제시 |
