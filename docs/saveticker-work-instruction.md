## 1. Execution protocol

### 1.1 Analysis phase (before any code changes)

```
1. Read `.claude/CLAUDE.md` for project conventions
2. Run `bun run typecheck` — record baseline error count
3. Run `bun run test` — record baseline test results
4. Read every file listed in Section 2.2 "Critical path files"
5. Enter EnterPlanMode and produce a sequenced plan
6. Get user approval before executing
```

### 1.2 Execution rules

- **One logical change per commit**. Never batch unrelated fixes.
- **Preserve all existing `data-label` attributes** — the project has 206 registered labels (see `docs/ui-labels.md`). Do not rename or remove any.
- **Bilingual JSDoc convention**: Every new or modified exported function/component must have `/** English / 한국어 */` JSDoc.
- **Convex naming**: All query exports prefixed with `get` (e.g., `getArticleById`). All tables are camelCase plural.
- **Feature folder convention**: kebab-case directories under `src/features/`. Hooks are one-per-file with `use` prefix.
- **Korean UI labels**: All user-facing text is Korean. English only in code comments and API names.
- **No new dependencies** unless explicitly approved below.
- After all changes: re-run `bun run typecheck` and `bun run test`. Both must pass with zero new errors.

convex/_generated/*          # Auto-generated by Convex
android/*                    # Native Android layer
src-ontology/*               # Legacy archived code (excluded from tsconfig)

## 4. P1 — Architecture & performance

### 4.1 Convex full-table scan in article queries

**File**: `convex/model/article.ts`

**Problem**: Three functions use a scan-then-filter pattern that reads up to 200 documents from the `by_publishedAt` index, then applies JS `.filter()`:

```
articlesByTicker()   — line ~65: .take(MAX_SCAN_LIMIT) then .filter(a => a.mentionedTickers?.includes(ticker))
articlesBySource()   — line ~97: .take(MAX_SCAN_LIMIT) then .filter(a => a.sourceName === sourceName)
articlesByTag()      — line ~114: .take(MAX_SCAN_LIMIT) then .filter(a => a.tags?.includes(tag))
```

**Fix for `articlesBySource`** — Use the Convex query builder with an index:

1. **Add index to `convex/schema.ts`** in the `newsArticles` table definition:

```typescript
// ADD this new index after the existing ones:
.index("by_sourceName", ["sourceName"])
```

2. **Rewrite `articlesBySource` in `convex/model/article.ts`**:

```typescript
export async function articlesBySource(
  ctx: QueryCtx,
  sourceName: string,
  limit: number = 50,
): Promise<Doc<"newsArticles">[]> {
  return await ctx.db
    .query("newsArticles")
    .withIndex("by_sourceName", (q) => q.eq("sourceName", sourceName))
    .order("desc")
    .take(limit);
}
```

**For `articlesByTicker` and `articlesByTag`** — Convex cannot index into arrays. The current scan+filter approach is acceptable for prototype scale (< 200 articles). Add a comment documenting this limitation:

```typescript
/**
 * Articles mentioning a specific ticker symbol.
 * 특정 티커 심볼을 언급하는 기사.
 *
 * ⚠ Convex cannot index into arrays — scans up to MAX_SCAN_LIMIT then filters in JS.
 * For production: denormalize into a join table (articleTickers) or use Convex search index.
 */
```

3. **After adding the index**, run `npx convex dev` to push the schema change (or note this in the plan as a deployment step).

### 4.3 `feedStore.ts` — Capacitor Preferences async storage with Zustand sync API mismatch

**File**: `src/stores/feedStore.ts`

**Problem**: The `createCapacitorStorage()` function returns async methods for native platform, but Zustand's `persist` middleware's `StateStorage` interface expects `getItem` to return `string | null | Promise<string | null>`. // ADD at module level, after the isNativePlatform check:
let _preferencesModule: typeof import("@capacitor/preferences") | null = null;

async function getPreferences() {
  if (!_preferencesModule) {
    _preferencesModule = await import("@capacitor/preferences");
  }
  return _preferencesModule;
}

//   const { Preferences } = await import("@capacitor/preferences");

#### 5.1.1 Replace category icon capsule with source initial avatar

Add a new helper function above the `NewsCard` component:

```typescript
/**
 * Source name initial + deterministic color for visual differentiation.
 * 출처 이름 이니셜 + 결정론적 색상.
 */
const sourceColors = [
  "bg-brand/15 text-brand",
  "bg-accent-1/15 text-accent-1",
  "bg-accent-2/15 text-accent-2",
  "bg-danger/15 text-danger",
  "bg-info/15 text-info",
  "bg-accent-4/15 text-accent-4",
];

function sourceColor(name: string): string {
  let hash = 0;
  for (let i = 0; i < name.length; i++) {
    hash = ((hash << 5) - hash) + name.charCodeAt(i);
    hash |= 0;
  }
  return sourceColors[Math.abs(hash) % sourceColors.length];
}
```

/**
 * Accordion section for explainer progressive disclosure.
 * 해설 점진적 공개를 위한 아코디언 섹션.
 */
import { useState, useId, type ReactNode } from "react";
import { motion, AnimatePresence } from "motion/react";
import { cn } from "../../../lib/cn";

interface ExplainerAccordionProps {
  /** Section title / 섹션 제목 */
  title: string;
  /** Whether section starts open / 초기 열림 상태 */
  defaultOpen?: boolean;
  /** Section content / 섹션 내용 */
  children: ReactNode;
  /** Title text color override / 제목 색상 오버라이드 */
  titleClassName?: string;
  /** Additional class names / 추가 클래스 */
  className?: string;
}

export function ExplainerAccordion({
  title,
  defaultOpen = false,
  children,
  titleClassName,
  className,
}: ExplainerAccordionProps) {
  const [isOpen, setIsOpen] = useState(defaultOpen);
  const contentId = useId();
  const headerId = useId();

  return (
    <div
      data-label="explainer.accordion"
      className={cn("border-t border-white/[0.06]", className)}
    >
      <button
        type="button"
        id={headerId}
        aria-expanded={isOpen}
        aria-controls={contentId}
        onClick={() => setIsOpen((prev) => !prev)}
        className="flex w-full items-center justify-between py-3 text-left min-h-[44px]"
      >
        <span className={cn("text-xs font-medium", titleClassName ?? "text-ink/70")}>
          {title}
        </span>
        <svg
          className={cn(
            "size-3.5 text-ink-muted/40 transition-transform duration-200",
            isOpen && "rotate-180",
          )}
          fill="none"
          viewBox="0 0 24 24"
          stroke="currentColor"
          strokeWidth={2}
        >
          <path strokeLinecap="round" strokeLinejoin="round" d="m19 9-7 7-7-7" />
        </svg>
      </button>
      <AnimatePresence initial={false}>
        {isOpen && (
          <motion.div
            id={contentId}
            role="region"
            aria-labelledby={headerId}
            initial={{ height: 0, opacity: 0 }}
            animate={{ height: "auto", opacity: 1 }}
            exit={{ height: 0, opacity: 0 }}
            transition={{ duration: 0.2, ease: [0.25, 0.1, 0.25, 1] }}
            className="overflow-hidden"
          >
            <div className="pb-3">{children}</div>
          </motion.div>
        )}
      </AnimatePresence>
    </div>
  );
}

### 8.4 Create stat extraction utility

**New file**: `src/features/explainer/lib/extractStats.ts`

```typescript
/**
 * Extract key statistics from explainer content for stat card display.
 * 통계 카드 표시를 위해 해설 콘텐츠에서 핵심 통계를 추출합니다.
 *
 * For prototype: regex-based extraction from Korean text patterns.
 * Production: these would be structured fields on the Explainer entity.
 */

interface StatItem {
  value: string;
  label: string;
  colorClass?: string;
}

/** Common number patterns in Korean financial text / 한국어 금융 텍스트의 일반적인 숫자 패턴 */
const PATTERNS: Array<{
  regex: RegExp;
  label: (match: RegExpMatchArray) => string;
  colorClass: string;
}> = [
  {
    regex: /(\d+(?:\.\d+)?)\s*%\s*(?:급등|폭등|상승|급락|폭락|하락)/,
    label: (m) => m[0].includes("등") ? "급등" : "급락",
    colorClass: "text-danger",
  },
  {
    regex: /\$[\d,.]+(?:\s*[조억만])?/,
    label: () => "금액",
    colorClass: "text-warning",
  },
  {
    regex: /(\d+)척.*(?:에서|→).*(\d+)척/,
    label: () => "유조선/일",
    colorClass: "text-danger",
  },
  {
    regex: /(\d+(?:\.\d+)?)\s*%.*의존/,
    label: () => "의존도",
    colorClass: "text-accent-5",
  },
];

export function extractStats(storyBody: string, keyTakeaways: string[]): StatItem[] {
  const allText = [storyBody, ...keyTakeaways].join(" ");
  const stats: StatItem[] = [];

  for (const pattern of PATTERNS) {
    if (stats.length >= 3) break;
    const match = allText.match(pattern.regex);
    if (match) {
      stats.push({
        value: match[0].length > 12 ? match[0].slice(0, 12) : match[0],
        label: pattern.label(match),
        colorClass: pattern.colorClass,
      });
    }
  }

  return stats;
}
```

### 8.5 Rewrite `ExplainerTab.tsx` with progressive disclosure

**File**: `src/features/explainer/components/ExplainerTab.tsx`

**Replace the entire component** with:

```typescript
/**
 * Full explainer view with progressive disclosure.
 * 점진적 공개 방식의 전체 해설 뷰.
 *
 * Layout:
 * 1. Difficulty badge + reading time estimate
 * 2. Title
 * 3. Stat cards (extracted key numbers)
 * 4. Accordion: 한줄 요약 (first 2 sentences of storyBody, default open)
 * 5. Accordion: 핵심 정리 (keyTakeaways)
 * 6. Accordion: 쉽게 비유하자면 (analogy, if present)
 * 7. Accordion: 나에게 어떤 영향? (personalImpact, highlighted)
 */
import type { Doc } from "../../../../convex/_generated/dataModel";
import { cn } from "../../../lib/cn";
import { Badge } from "../../../components/ui/Badge";
import { ExplainerAccordion } from "./ExplainerAccordion";
import { ExplainerStatCards } from "./ExplainerStatCards";
import { extractStats } from "../lib/extractStats";

interface ExplainerTabProps {
  explainer: Doc<"explainers">;
  className?: string;
}

const difficultyBadge: Record<string, { label: string; variant: "success" | "warning" | "danger" }> = {
  beginner: { label: "초급", variant: "success" },
  intermediate: { label: "중급", variant: "warning" },
  advanced: { label: "고급", variant: "danger" },
};

/**
 * Estimate reading time in minutes from Korean text.
 * 한국어 텍스트에서 읽기 시간을 분 단위로 추정합니다.
 * Average Korean reading speed: ~400 characters/minute.
 */
function estimateReadingTime(explainer: Doc<"explainers">): number {
  const totalChars = (
    (explainer.storyBody?.length ?? 0) +
    (explainer.keyTakeaways?.join("").length ?? 0) +
    (explainer.personalImpact?.length ?? 0) +
    (explainer.analogy?.length ?? 0)
  );
  return Math.max(1, Math.round(totalChars / 400));
}

export function ExplainerTab({ explainer, className }: ExplainerTabProps) {
  const difficulty = difficultyBadge[explainer.difficultyLevel] ?? {
    label: explainer.difficultyLevel,
    variant: "success" as const,
  };
  const readingTime = estimateReadingTime(explainer);
  const stats = extractStats(explainer.storyBody, explainer.keyTakeaways);

  return (
    <article
      data-label="explainer.tab"
      className={cn("rounded-xl border glass-panel p-5 shadow-md animate-in space-y-4", className)}
    >
      {/* Header: difficulty badge + reading time / 헤더: 난이도 뱃지 + 읽기 시간 */}
      <div data-label="explainer.tab.header" className="flex items-center gap-2">
        <Badge variant={difficulty.variant}>{difficulty.label}</Badge>
        <span className="text-[10px] text-ink-muted/50">읽기 {readingTime}분</span>
      </div>

      {/* Title / 제목 */}
      <h3 data-label="explainer.tab.title" className="text-base font-medium text-ink">
        {explainer.simplifiedTitle}
      </h3>

      {/* Stat cards / 통계 카드 */}
      {stats.length > 0 && <ExplainerStatCards stats={stats} />}

      {/* Accordion sections / 아코디언 섹션 */}

      {/* Story body — default open / 스토리 본문 — 기본 열림 */}
      <ExplainerAccordion title="한줄 요약" defaultOpen>
        <p data-label="explainer.tab.body" className="text-sm leading-relaxed text-ink/90 whitespace-pre-wrap">
          {explainer.storyBody}
        </p>
      </ExplainerAccordion>

      {/* Key takeaways / 핵심 정리 */}
      {explainer.keyTakeaways.length > 0 && (
        <ExplainerAccordion title={`핵심 정리 (${explainer.keyTakeaways.length})`}>
          <ul data-label="explainer.tab.takeaways" className="space-y-2">
            {explainer.keyTakeaways.map((takeaway, i) => (
              <li key={i} className="flex items-start gap-2 text-sm text-ink/90">
                <span className="mt-1.5 size-1.5 shrink-0 rounded-full bg-accent-1" />
                {takeaway}
              </li>
            ))}
          </ul>
        </ExplainerAccordion>
      )}

      {/* Analogy / 비유 */}
      {explainer.analogy && (
        <ExplainerAccordion title="쉽게 비유하자면">
          <div data-label="explainer.tab.analogy" className="rounded-xl border glass-panel p-3">
            <p className="text-sm italic text-ink/80">{explainer.analogy}</p>
          </div>
        </ExplainerAccordion>
      )}

      {/* Personal impact — highlighted / 나에게 어떤 영향? — 강조 */}
      {explainer.personalImpact && (
        <ExplainerAccordion
          title="나에게 어떤 영향?"
          titleClassName="text-warning/80"
        >
          <div data-label="explainer.tab.personalImpact" className="rounded-xl border border-warning/10 bg-warning/[0.03] p-3">
            <p className="text-sm text-ink/85">{explainer.personalImpact}</p>
          </div>
        </ExplainerAccordion>
      )}
    </article>
  );
}
```
